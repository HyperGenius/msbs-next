# プロジェクトドキュメント管理ガイド

このディレクトリは、本プロジェクトの設計、仕様、運用に関する全ての資料を管理します。

## ディレクトリ構成と役割

| ディレクトリ | 役割 | 入れるべきドキュメントの例 |
| :--- | :--- | :--- |
| `architecture/` | システム全体の設計・思想 | 全体構成図、データフロー、認証フロー |
| `features/` | 各機能の要件・仕様 | バトルエンジンの仕様、バッチ処理の定義 |
| `implementations/`| 実装の詳細・技術選定 | 特定ライブラリの使用法、APIの実装詳細 |
| `infrastructure/` | インフラ・デプロイ | GCP構成、DB移行、CI/CDの設定 |
| `roadmap/` | 開発計画 | 短期/長期ロードマップ、UI改善案 |
| `setup/` | 開発準備・ガイド | 開発環境構築手順、テスト実行ガイド |
| `ui-design/` | デザイン・UI仕様 | デザインガイドライン、モックアップHTML |

## 運用ルール

### 1. ファイルの命名規則
- 全て **ケバブケース (`kebab-case.md`)** で作成してください。
- スペルミスを防ぐため、作成後は必ずファイル名を確認してください。

### 2. ドキュメントの更新
- 実装に変更があった場合、関連するドキュメントも速やかに更新してください。
- 古くなったドキュメントには冒頭に `> [!CAUTION] 内容が古い可能性があります` と追記するか、削除を検討してください。

### 3. 画像や図解
- ドキュメント内で使用する画像は、各ディレクトリ内の `assets/` フォルダに集約してください。
