# パラメータチューニング UI 抜本改善提案

**バージョン**: 1.1.0  
**作成日**: 2026-05-31  
**最終更新**: 2026-05-31  
**ステータス**: Phase 1 + Phase 2 実装済み（Issue #375）  
**対象コンポーネント**: `frontend/src/components/pilot/ParameterTuningPanel.tsx`

---

## 1. 概要と目的

パイロットのステータスポイント割り振り画面（PARAMETER TUNING）は現状、6 種のステータスカードを単列縦積みで表示するシンプルな構成になっている。この構成では以下の問題が生じており、ゲームプレイ上の意思決定を妨げている。

本ドキュメントは現状の問題点を整理し、抜本的なリデザインの方向性と具体的な仕様を示す。

---

## 2. 現状の課題

### 2.1 問題点一覧

| # | カテゴリ | 問題の説明 | 影響度 |
|---|----------|-----------|--------|
| 1 | レイアウト | 6 カードが縦 1 列に並んでおり、全ステータスを一覧しにくい | 高 |
| 2 | 可視化 | 数値とテキストのみ。グラフ・プログレスバーが一切ない | 高 |
| 3 | 情報設計 | `Rank D(3)` の `(3)` が「現在値」なのか「保留中の増分」なのか一見不明 | 高 |
| 4 | 情報の欠落 | `STATUS_LABELS.desc`（各ステータスのゲーム効果説明）がコード上に定義済みだが UI に表示されていない | 中 |
| 5 | ポイント残量 | 「未割り振りポイント: 5」という小テキスト 1 行のみで直感的でない | 中 |
| 6 | 操作性 | 保留中の割り当てを一括リセットする手段がない | 中 |
| 7 | 配色 | ランク文字以外は全て同色系（暗緑・黒）でメリハリがなく疲れる | 低 |

### 2.2 現状のコンポーネント構造

```
ParameterTuningPanel
├── 未割り振りポイントテキスト（1行）
├── StatCard × 6（縦積み）
│   ├── 略称 / ラベル　　Rank X(N)
│   ├── [-] ボタン
│   └── [+] ボタン
└── [SAVE PARAMETERS (N pt)] ボタン
```

### 2.3 現状の表示定義（コード内）

コード上には改善に活用できる情報がすでに定義されており、これらは今後の改善で積極的に活用すべきである。

## 3. 改善方針

### 3.1 設計思想

> **「数字を見る画面」から「ビルドを考える画面」へ**

- プレイヤーが自分のパイロットの「強さの形」を視覚的に把握できること
- どのステータスをどれだけ上げると何が変わるかを直感的に理解できること
- 操作ミスを防ぎ、やり直しが容易なこと

### 3.2 優先度分類

| 優先度 | 内容 |
|--------|------|
| **P0 必須** | 情報設計の修正（保留表示の曖昧さ解消、説明文の表示） |
| **P1 重要** | レーダーチャートの追加、レイアウト刷新 |
| **P2 改善** | ポイントゲージ再設計、ステータスバー追加 |
| **P3 細部** | リセットボタン、次ランクヒント、トースト通知 |

## 4. 各改善案の詳細仕様

### 4.1 レイアウト刷新

#### 現状
```
[縦 1 列にカード 6 枚]
```

#### 提案レイアウト（2 ペイン構成）

```
┌────────────────────────────────────────────────┐
│  PARAMETER TUNING                              │
├──────────────────┬─────────────────────────────┤
│                  │  ┌──────┐  ┌──────┐         │
│  [レーダーチャート]  │  │ SHT  │  │ MEL  │         │
│                  │  └──────┘  └──────┘         │
│  ◆ 残ポイント     │  ┌──────┐  ┌──────┐         │
│    ゲージ         │  │ INT  │  │ REF  │         │
│                  │  └──────┘  └──────┘         │
│                  │  ┌──────┐  ┌──────┐         │
│                  │  │ TOU  │  │ LUK  │         │
│                  │  └──────┘  └──────┘         │
├──────────────────┴─────────────────────────────┤
│  [RESET]                [SAVE PARAMETERS (N pt)]│
└────────────────────────────────────────────────┘
```

- **左ペイン**：レーダーチャート + ポイントゲージ（幅 40%）
- **右ペイン**：3×2 グリッドのステータスカード（幅 60%）
- **フッター**：リセット + 保存ボタン

モバイル（sm 以下）では現状の縦積みにフォールバック（チャートを上部に配置）。

---

### 4.2 レーダーチャート（Spider Chart）

#### 目的
6 ステータスの相対的な強さを一目で把握できるようにする。

#### 仕様

| 項目 | 内容 |
|------|------|
| 形状 | 正六角形（軸: SHT / MEL / INT / REF / TOU / LUK） |
| スケール | 0〜25（最大値） |
| 表示レイヤー | 現在値（実線 + 塗り、`#00ff41` 20% 透過） |
| 表示レイヤー | 保留後予測値（破線 + 塗りなし、`#00f0ff`） |
| アニメーション | 保留変更時に予測値レイヤーが滑らかに変形（200ms ease） |
| 実装ライブラリ候補 | recharts `RadarChart` または SVG 直描き（依存追加を避けたい場合） |

#### モックアップ（ASCII）

```
          SHT
         /    \
    LUK ──●────── MEL
        / ■■■■ \
   TOU ──■■■■■──── INT
        \  ■■  /
         \    /
          REF

■ = 現在値（緑塗り）
● = 保留後（シアン破線）
```

#### 実装上の注意
- SSR 対応のため `dynamic import` で `ssr: false` を設定する
- `recharts` はすでにプロジェクトに導入されているか確認すること（未導入の場合は SVG 直描きを推奨）

### 4.3 ステータスカード再設計

#### 現状カード

```
┌──────────────────────────────────┐
│ SHT / 射撃精度    Rank D(3)  (+3) │
│ [        -        ] [     +     ] │
└──────────────────────────────────┘
```

#### 提案カード

```
┌─────────────────────────────────┐
│ SHT  射撃精度                    │
│ 射撃攻撃力補正率（シグモイド）      │
│                                 │
│ ██████░░░░░░░░░░░░░░  3pt       │
│ D ──────────────────── C  B  A  S│
│                                 │
│ 現在: D(3)  +3  →  保留後: D(6) │
│                                 │
│       [-]              [+]      │
└─────────────────────────────────┘
```

#### 各要素の仕様

| 要素 | 仕様 |
|------|------|
| ヘッダー | 略称（太字）+ 日本語名 |
| 説明文 | `STATUS_LABELS[stat].desc` をグレーテキストで表示 |
| プログレスバー | 現在値を `0〜25` スケールで表示。色はランク色（`STAT_RANK_COLORS`）と連動 |
| プログレスバー（保留） | 保留分を別色（シアン `#00f0ff`）の重ね表示で示す |
| ランク閾値マーカー | バー下に `D C B A S` の閾値ラインを表示 |
| 値の変化表示 | `現在: D(3) +3 → 保留後: D(6)` の形式で明確に分離 |
| ボタン | 既存の `SciFiButton` を再利用。`-` は pending が 0 の場合は disabled |

#### ランクカラー連動ルール

```typescript
// バーの色: 保留後ランクに応じて変化
const barColor = STAT_RANK_COLORS[getStatRank(currentValue + pending)];
```

### 4.4 ポイントゲージ再設計

#### 現状
```
未割り振りポイント: 5
```

#### 提案：バッテリー型ゲージ

```
┌─────────────────────────────────┐
│  STATUS POINTS                  │
│  ████████████░░░░░   5 / 8 pt   │
│  ▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░             │
└─────────────────────────────────┘
```

| 残ポイント割合 | ゲージ色 |
|--------------|---------|
| 67% 以上 | `#00ff41`（緑） |
| 34〜66% | `#ffb000`（アンバー） |
| 1〜33% | `#ff8800`（オレンジ） |
| 0% | `#ff4444`（赤）+ 警告パルスアニメーション |

- ゲージ幅：保留後の残量をリアルタイム反映
- 数値表示：`残 N pt / 合計 M pt`（保留後の数字を表示）
- 0 になった際に `+` ボタン全体を無効化（現状通り）

### 4.5 UX 細部改善

#### 4.5.1 リセットボタン

- 位置：フッター左側
- 動作：`pendingAlloc` を全て 0 にリセット
- 条件：保留が 1 つ以上ある場合のみ有効

```typescript
const hasAnyPending = Object.values(pendingAlloc).some(v => v > 0);
```

#### 4.5.2 次ランクまでのヒント

各カード内に以下の形式で表示：

```
次のランク（C）まであと 2 pt
```

```typescript
function ptToNextRank(current: number, pending: number): { rank: string; pt: number } | null {
  const thresholds = [5, 10, 15, 20]; // C, B, A, S
  const total = current + pending;
  const next = thresholds.find(t => t > total);
  if (!next) return null;
  return { rank: getStatRank(next), pt: next - total };
}
```

#### 4.5.3 保存後のフィードバック

現状は `allocSuccess` テキストが表示されるのみ。以下に改善：

- 保存成功後にトースト通知（3 秒後自動消滅）を表示
- トーストに「元に戻す」ボタンを設け、前回の値への再割り当てリクエストを送信
- トースト実装は既存の `SciFiPanel variant="accent"` を流用

## 7. 実装優先度ロードマップ

### Phase 1 — 情報設計の修正（P0）

> 最小変更でユーザーの混乱を即時解消する

- [ ] 保留表示の形式変更：`Rank D(3)` → `現在: D(3) +3 → 保留後: D(6)`
- [ ] `STATUS_LABELS.desc` の説明文をカード内に表示
- [ ] `allocSuccess`/`allocError` をトースト形式に変更

> プレイヤーの意思決定品質を向上させる視覚的改善

- [ ] レーダーチャートの追加（`StatusRadarChart.tsx` 新規作成）

- [ ] ポイントゲージの再設計（`StatusPointGauge.tsx` 新規作成）
- [ ] レイアウト 2 ペイン化

> 操作快適性の向上

- [ ] リセットボタンの追加
