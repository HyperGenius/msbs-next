# MSBS UI/UX ガイドライン (Mobile First)

本ドキュメントは、MSBSにおけるユーザー体験（`docs/specs/CORE_LOOP.md` 参照）を最大限に引き出すための、画面設計・レイアウトの基本原則を定めます。

## 1. 基本方針：モバイルファーストと「1回5分」の最適化
ターゲットユーザーが隙間時間にスマートフォンで快適にプレイできるよう、PC向けの情報を単に縮小するのではなく、スマートフォンでの身体的・認知的な使いやすさを最優先して設計します。

## 2. レイアウトの3原則

### ① 「親指の導線（Thumb Zone）」を意識した配置
スマートフォンの操作は基本的に片手の親指で行われることを前提とします。
- **主要アクションのボトム配置**: 「装備確定」「出撃」「タブ切り替え」などの頻繁に行う、あるいは重要なアクションボタンは、画面下部（ボトムナビゲーションや固定フッター）に配置する。
- **ヘッダーの軽量化**: 画面上部（ヘッダー）は「戻る」ボタンや現在位置の表示など、補助的な役割に留める。

### ② 情報の段階的開示（Progressive Disclosure）
5インチ前後の狭い画面で情報過多によるストレス（認知負荷）を与えないよう、情報を要約と詳細に分割します。
- **ファーストビューの整理**: 画面を開いた瞬間に目に入る情報は、「次回の戦場環境」や「現在の総合ステータス」など、直感的に状況がわかるサマリー（要約）に絞る。
- **詳細情報の隠蔽と展開**: 詳細なパラメータや長いリストは、タップ（アコーディオン）やスワイプ、タブ切り替えによって、ユーザーが「見たい」と思ったタイミングで開示するUIとする。

### ③ タップ領域（Touch Target）の確実な確保
独自の世界観（Sci-Fi）を表現するシャープなデザインを維持しつつも、操作ミス（誤タップ）を防ぐための余白を確保します。
- **最小タップ領域**: 全てのボタンやインタラクティブな要素は、視覚的なデザインの大小に関わらず、クリック判定領域（padding等を含む）を **最低 44px × 44px** 確保する。
- （実装の詳細については `SF_UI_COMPONENTS_GUIDE.md` を参照）

## 3. レスポンシブ設計のブレイクポイント
Tailwind CSSの標準ブレイクポイントを利用し、以下のように画面を拡張します。
- `デフォルト (スマホ)`: 1カラムレイアウト。タブやアコーディオンを駆使して情報を整理。
- `md (タブレット)`: 余白の調整。一部のアコーディオンをデフォルトで展開。
- `lg (PC)`: スマホ向けレイアウトをベースにしつつ、余った左右の空間を利用して「リスト」と「詳細」を2カラムで並べて表示するなど、広い画面を活かした配置へのプログレッシブ・エンハンスメントを行う。
